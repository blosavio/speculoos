Valid versus Validation

Validation is the process of systematically applying predicates to scalars and collections. The validation process returns an exhaustive sequence of every datum-predicate pair and the truthy/falsey result of the validation.

(require '[speculoos.core :refer
           [validate-scalars valid-scalars? only-valid
            only-invalid]])
(validate-scalars [42 :foo "abc"] [int? keyword? boolean?])
;; => [{:datum 42,
;;      :path [0],
;;      :predicate #function [clojure.core/int?],
;;      :valid? true}
;;     {:datum :foo,
;;      :path [1],
;;      :predicate #function [clojure.core/keyword?],
;;      :valid? true}
;;     {:datum "abc",
;;      :path [2],
;;      :predicate #function [clojure.core/boolean?],
;;      :valid? false}]

Speculoos provides some helper functions to filter the results.

(only-valid (validate-scalars [42 :foo "abc"]
                              [int? keyword? boolean?]))
;; => ({:datum 42,
;;      :path [0],
;;      :predicate #function [clojure.core/int?],
;;      :valid? true}
;;     {:datum :foo,
;;      :path [1],
;;      :predicate #function [clojure.core/keyword?],
;;      :valid? true})
(only-invalid (validate-scalars [42 :foo "abc"]
                                [int? keyword? boolean?]))
;; => ({:datum "abc",
;;      :path [2],
;;      :predicate #function [clojure.core/boolean?],
;;      :valid? false})

Valid is a high level true/false summary of a validation. If the only-invalids sequence is empty, then valid? is true; false otherwise.

(valid-scalars? [42 :foo "abc"] [int? keyword? string?])
;; => true
(valid-scalars? [42 :foo "abc"] [int? keyword? boolean?])
;; => false

Scalar validation

Scalar validation applies predicates to basic Clojure datums like numbers, strings, booleans, characters, keywords, etc.

Empty collections

Speculoos can validate in the absence of scalars. Predicates are simply ignored.

(validate-scalars [] [int?]) ;; => [](validate-scalars '() [int?]) ;; => [](validate-scalars {} [int?]) ;; => [](validate-scalars #{} [int?]) ;; => []

Empty specifications

Validating may also proceed without predicates. Not much happens.

(validate-scalars [42 :foo "abc"] []) ;; => [](validate-scalars '(42 :foo "abc") []) ;; => [](validate-scalars {:a 42, :b :foo, :c "abc"} []) ;; => [](validate-scalars #{42 :foo "abc"} []) ;; => []

Vectors

Speculoos validates a vector's scalars in order.

(valid-scalars? [42] [int?]) ;; => true(valid-scalars? [42 :foo] [int? keyword?]) ;; => true(valid-scalars? [42 :foo "abc"] [int? keyword? string?])
;; => true
(only-invalid (validate-scalars [42 :foo "abc"]
                                [int? keyword? boolean?]))
;; => ({:datum "abc",
;;      :path [2],
;;      :predicate #function [clojure.core/boolean?],
;;      :valid? false})

Lists

Speculoos validates a list's scalars in order.

(valid-scalars? (list 42) (list int?)) ;; => true(valid-scalars? (list 42 :foo) (list int? keyword?))
;; => true(valid-scalars? (list 42 :foo "abc")
                (list int? keyword? string?))
;; => true
(only-invalid (validate-scalars
                (list 42 :foo "abc")
                (list int? keyword? boolean?)))
;; => ({:datum "abc",
;;      :path [2],
;;      :predicate #function [clojure.core/boolean?],
;;      :valid? false})

Maps

Speculoos validates a map's scalars by key.

(valid-scalars? {:a 42} {:a int?}) ;; => true(valid-scalars? {:a 42, :b 'foo} {:a int?, :b symbol?})
;; => true
(valid-scalars? {:a 42, :b 'foo, :c "abc"}
                {:a int?, :b symbol?, :c string?})
;; => true
(valid-scalars? {:a 42, :b 'foo, :c "abc"}
                {:b symbol?, :c string?, :a int?})
;; => true
(only-invalid (validate-scalars
                {:a 42, :b 'foo, :c "abc"}
                {:a int?, :b boolean?, :c string?}))
;; => ({:datum foo,
;;      :path [:b],
;;      :predicate #function [clojure.core/boolean?],
;;      :valid? false})

Sets

Speculoos validates a set's scalars by applying all predicates to every scalar. If all predicates return true for every scalar, then the data and specification are valid. If any scalar does not satisfy a single predicate, then it's invalid.

(valid-scalars? #{42} #{int?}) ;; => true(valid-scalars? #{42 0 99} #{int?}) ;; => true(valid-scalars? #{42 'foo 99} #{int?}) ;; => false
(valid-scalars? #{42} #{int? number? pos?}) ;; => true(valid-scalars? #{42} #{int? string?}) ;; => false(valid-scalars? #{42 99 -11} #{int? number? pos?})
;; => false
(defn int-or-str? [x] ((some-fn int? string?) x))
(valid-scalars? #{42 "abc" 99} #{int-or-str?}) ;; => true

Non-terminating sequences

Speculoos can handle non-terminating sequences in two situations. First, Speculoos can validate a non-terminating sequence of scalars as long as the corresponding specification is not also a non-terminating sequence. Validation will proceed until the predicates run out.

(valid-scalars? (repeat 42) [int?]) ;; => true(valid-scalars? (repeat 42) [int? int? int?]) ;; => true
(only-invalid (validate-scalars (repeat 42)
                                [int? string? int?]))
;; => ({:datum 42,
;;      :path [1],
;;      :predicate #function [clojure.core/string?--5475],
;;      :valid? false})
(valid-scalars? (cycle [42 :foo "abc"]) [int?]) ;; => true
(valid-scalars? (cycle [42 :foo "abc"])
                [int? keyword? string?])
;; => true
(only-invalid (validate-scalars (cycle [42 :foo "abc"])
                                [int? boolean? string?]))
;; => ({:datum :foo,
;;      :path [1],
;;      :predicate #function [clojure.core/boolean?],
;;      :valid? false})
(valid-scalars? (range) [int?]) ;; => true(valid-scalars? (range) [int? int? int?]) ;; => true(valid-scalars? (range) [int? boolean? int?]) ;; => false
(valid-scalars? (iterate inc 1) [int?]) ;; => true(valid-scalars? (iterate inc 1) [int? int? int?]) ;; => true
(valid-scalars? (iterate inc 1) [int? boolean? int?])
;; => false
(valid-scalars? (lazy-seq [1 2 3]) [int?]) ;; => true(valid-scalars? (lazy-seq [1 2 3]) [int? int? int?])
;; => true
(valid-scalars? (lazy-seq [1 2 3]) [int? boolean? int?])
;; => false

Second, Speculoos can use a specification that is composed of a non-terminating sequence as long as the data is not also a non-terminating sequence.

(valid-scalars? [42] (repeat int?)) ;; => true(valid-scalars? [42 0 99] (repeat int?)) ;; => true
(valid-scalars? [42 :foo "abc"]
                (cycle [int? keyword? string?]))
;; => true
(valid-scalars? [42 :foo "abc"]
                (cycle [int? boolean? string?]))
;; => false

Nested collections

Speculoos can validate any scalar in a heterogeneous, arbitrarily-nested data structure. The scalar and predicate share a path into the data and specification, respectively. Another way to think about it is that the specification mimics the data's structure, and the predicate and scalar are located at the same position within each.

Examples of homogeneous, nested collections.

(valid-scalars? [42 [:foo] "abc"] [int? [keyword?] string?])
;; => true
(valid-scalars? [42 [:foo ["abc"]]]
                [int? [keyword? [string?]]])
;; => true
(valid-scalars? [[42] [:foo] ["abc"]]
                [[int?] [keyword?] [string?]])
;; => true
(valid-scalars? (list 42 (list :foo (list "abc")))
                (list int? (list keyword? (list string?))))
;; => true
(valid-scalars? {:a 42, :b {:c 'foo, :d "abc"}}
                {:a int?, :b {:c symbol?, :d string?}})
;; => true
(valid-scalars? {:a 42, :b {:c {:d 'foo, :e "abc"}}}
                {:a int?, :b {:c {:d symbol?, :e string?}}})
;; => true

Examples of heterogeneous, nested collections.

(validate-scalars [42 {:a 'foo, :b "abc"}]
                  [int? {:a symbol?, :b string?}])
;; => [{:datum 42,
;;      :path [0],
;;      :predicate #function [clojure.core/int?],
;;      :valid? true}
;;     {:datum foo,
;;      :path [1 :a],
;;      :predicate #function [clojure.core/symbol?],
;;      :valid? true}
;;     {:datum "abc",
;;      :path [1 :b],
;;      :predicate #function [clojure.core/string?--5475],
;;      :valid? true}]
(validate-scalars [42 {:a ['foo {:b (list "abc")}]}]
                  [int? {:a [symbol? {:b (list string?)}]}])
;; => [{:datum 42,
;;      :path [0],
;;      :predicate #function [clojure.core/int?],
;;      :valid? true}
;;     {:datum foo,
;;      :path [1 :a 0],
;;      :predicate #function [clojure.core/symbol?],
;;      :valid? true}
;;     {:datum "abc",
;;      :path [1 :a 1 :b 0],
;;      :predicate #function [clojure.core/string?--5475],
;;      :valid? true}]
(validate-scalars [[[[42]]] [[[:foo]]] [[["abc"]]]]
                  [[[[int?]]] [[[keyword?]]] [[[string?]]]])
;; => [{:datum 42,
;;      :path [0 0 0 0],
;;      :predicate #function [clojure.core/int?],
;;      :valid? true}
;;     {:datum :foo,
;;      :path [1 0 0 0],
;;      :predicate #function [clojure.core/keyword?],
;;      :valid? true}
;;     {:datum "abc",
;;      :path [2 0 0 0],
;;      :predicate #function [clojure.core/string?--5475],
;;      :valid? true}]

Scalars contained in a nested set can be validated.

(valid-scalars? [#{42 0 99} #{'foo} #{:foo :bar :baz}]
                [#{int?} #{symbol?} #{keyword?}])
;; => true

Adjusting specifications

What if the specification in your hand is close, but not exactly, what you need? You can relax a specification with a couple of strategies. First strategy: Speculoos will only validate if it has a predicate, so you could simply remove the offending predicate.

(valid-scalars? [42 :foo "abc"] [int? keyword? boolean?])
;; => false
(valid-scalars? [42 :foo "abc"]
                (pop [int? keyword? boolean?]))
;; => true
(valid-scalars? {:a 42, :b 'foo} {:a int?, :b boolean?})
;; => false
(valid-scalars? {:a 42, :b 'foo}
                (dissoc {:a int?, :b boolean?} :b))
;; => true

Second strategy: Replace the offending predicate with a more permissive predicate.

(valid-scalars? [42 :foo "abc"] [int? keyword? boolean?])
;; => false
(valid-scalars? [42 :foo "abc"] [int? keyword? any?])
;; => true
(valid-scalars? {:a 42, :b 'foo} {:a int?, :b boolean?})
;; => false
(valid-scalars? {:a 42, :b 'foo}
                (assoc {:a int?, :b boolean?} :b any?))
;; => true

If you need to dive into a nested structure to adjust the specification, Speculoos has a few functions that'll help. Speculoos also provides a few utilities specifically for adjusting specifications.

Repairing data

What if you have a specification, and some data that almost satisfies it? You could adjust the data so that it satisfies the specification. The crude way would be to remove the offending datum.

(valid-scalars? {:a 42, :b 'foo} {:a int?, :b string?})
;; => false
(valid-scalars? (dissoc {:a 42, :b 'foo} :b)
                {:a int?, :b string?})
;; => true

Another option involves associating a new value to satisfy the predicate.

(valid-scalars? (assoc {:a 42, :b 'foo} :b "foo")
                {:a int?, :b string?})
;; => true

Optionality

Speculoos provides a handful of ways to indicate optionality, depending on the situation. Speculoos does not infer nor assume; it only validates pairs of datums and predicates. Supplying predicates to a specification says These datums may or may not exist. If the datums exist, they must satisfy these predicates. If the datums do not exist, the predicates are ignored.

(valid-scalars? [42 :foo]
                [int? keyword? string? boolean? ratio?
                 char?])
;; => true
(valid-scalars? [42 :foo "abc" true]
                [int? keyword? string? boolean? ratio?
                 char?])
;; => true
(valid-scalars? {:a 42, :b 'foo}
                {:a int?,
                 :b symbol?,
                 :c string?,
                 :d boolean?,
                 :e ratio?,
                 :f char?})
;; => true
(valid-scalars? {:a 42, :b 'foo, :d true, :f \c}
                {:a int?,
                 :b symbol?,
                 :c string?,
                 :d boolean?,
                 :e ratio?,
                 :f char?})
;; => true

Within a sequence, if you'd like to validate some of the values, but skip over others, you have a few options to maintain the correspondence between the data and specification: any?, (constantly true), or (fn [_] true).

(valid-scalars? [42 :foo "abc"] [int? any? string?])
;; => true
(valid-scalars? [42 :foo "abc"]
                [int? (constantly true) string?])
;; => true
(valid-scalars? [42 :foo "abc"]
                [int? (fn [_] true) string?])
;; => true

The results are the same. Pick the option that best signals your intent.

Maps are easier: Any key in both the specification and the data will be validated. Otherwise, unpaired datums or predicates are ignored.

(valid-scalars? {:validated 42, :ignored-datum 'foo}
                {:validated int?, :ignored-predicate char?})
;; => true

Considering a single datum, you can bestow optionality by composing a predicate with or, or, my preference, clojure.core/some-fn.

(valid-scalars? [42 :foo "abc"]
                [int? #(or (symbol? %) (keyword? %))
                 (some-fn char? string?)])
;; => true

Collection validation

Collection validation applies predicates to Clojure data structures like vectors, lists, maps, sets, lazy sequences, etc.

Empty specifications

Speculoos will consume an empty specification, and return.

(require '[speculoos.core :refer
           [validate-collections valid-collections?]])
(validate-collections [] []) ;; => ()(validate-collections '() '()) ;; => ()(validate-collections {} {}) ;; => ()(validate-collections #{} #{}) ;; => ()

There is nothing particularly special about validating an empty collection. In fact, that's a pretty useful task.

(valid-collections? [] [empty?]) ;; => true(valid-collections? '() '(empty?)) ;; => true(valid-collections? {} {:s empty?}) ;; => true(valid-collections? #{} #{empty?}) ;; => true

Vectors

Collection predicates apply to the vectors that contain them.

(valid-collections? [99 []] [(complement empty?) [empty?]])
;; => true
(valid-collections? [1 2 3] [#(= 3 (count %))]) ;; => true
(defn length-1? [v] (= 1 (count v)))
(valid-collections? [[1] [2] [3]]
                    [[length-1?] [length-1?] [length-1?]])
;; => true

Extending that last example: intervening collection predicates all apply to the root, parent vector.

(valid-collections? [[1] [2] [3]]
                    [vector? [length-1?] (complement empty?)
                     [length-1?] any? [length-1?]])
;; => true
(valid-collections? [[1] [2] [3]]
                    [[length-1?] vector? (complement empty?)
                     [length-1?] any? [length-1?]])
;; => true
(valid-collections? [[1] [2] [3]]
                    [[length-1?] [length-1?] any?
                     (complement empty?) vector?
                     [length-1?]])
;; => true

As seen above, any number of predicates apply to their parent collection, and in any order.

Lists

Speculoos validates lists exactly the same as vectors, so everything above holds for lists. A few quick examples.

(valid-collections? '() '(list?)) ;; => true
(valid-collections? '(1 2 3) '(#(= 3 (count %)))) ;; => true
(valid-collections? '((1) (2) (3))
                    '((length-1?) (length-1?) (length-1?)))
;; => true
(valid-collections?
  '((1) (2) (3))
  '((length-1?) list? (length-1?) (length-1?)))
;; => true

Maps

Collection predicates apply to the maps that contain them, as long as the key does not appear in the data's corresponding map. Any number of predicates may be supplied. Since the specification's keys don't matter, you can name them something informative.

(valid-collections? {:a 42} {:map-spec empty?}) ;; => false
(valid-collections? {:a 42} {:coll-test map?}) ;; => true
(valid-collections? {:a 42}
                    {:coll-test map?,
                     :coll-size #(= 1 (count %))})
;; => true
(valid-collections? {:a 42} {:has-key? #(contains? % :a)})
;; => true
(valid-collections? {:a 42, :b 'foo, :c 99}
                    {:all-vals-ints? #(every? int?
                                              (vals %))})
;; => false

Sets

Predicates within a set apply to the set itself as a whole.

(valid-collections? #{1 2 3} #{#(every? int? %)}) ;; => true
(valid-collections? #{1 2 :foo} #{#(every? int? %)})
;; => false
(valid-collections? #{1 2 3}
                    #{#(every? int? %) #(every? pos? %)})
;; => true
(valid-collections? #{1 2 3}
                    #{#(every? int? %) #(every? even? %)})
;; => false

All elements not something.

(valid-collections? #{1 2 3}
                    #{#(every? (complement keyword?) %)})
;; => true

For every vector contained in a set, does it contain one integer?

(valid-collections?
  #{:a [1] :b [42] :c [99]}
  #{(fn [s] (every? #(= 1 (count %)) (filter vector? s)))})
;; => true
(valid-collections?
  #{:a [1 2 3] :b}
  #{(fn [s] (every? #(= 1 (count %)) (filter vector? s)))})
;; => false

Non-terminating Sequences

You may supply a (possibly) non-terminating sequence in the data, or in the specification, but not both. During collection validation, the non-terminating sequence is clamped to a length of the other corresponding sequence.

(set! *print-length* 99) ;; => 99
(valid-collections? (range 99) [#(every? int? %)])
;; => true
(valid-collections? (range 99) [#(every? ratio? %)])
;; => false
(valid-collections?
  (cycle [42 :foo "abc"])
  [#(valid-scalars? % [int? keyword? string?])])
;; => true
(valid-collections?
  (iterate #(+ 3 %) 1)
  [(fn [s] (every? true? (map #(= (+ 3 %1) %2) s (next s))))
   (constantly true) (constantly true)])
;; => true
(valid-collections? (repeat :na)
                    [(fn [s] (every? #(= :na %) s))
                     (constantly true) (constantly true)])
;; => true

Nested collections

Same rule, recursively: predicates apply to their parent collection.

(valid-collections? [[1] [:a] ['foo]]
                    [[#(every? int? %)]
                     [#(every? keyword? %)]
                     [#(every? symbol? %)]])
;; => true
(valid-collections? [[1 22/7] [:a] ['foo]]
                    [[#(every? int? %)]
                     [#(every? keyword? %)]
                     [#(every? symbol? %)]])
;; => false
(valid-collections? [11 [22 33 44]]
                    [vector? [#(= 3 (count %))]
                     #(= 2 (count %))])
;; => true
(valid-collections?
  {:a 11, :b {:c 22}}
  {:has-keys? #(and (contains? % :a) (contains? % :b)),
   :b {:is-map? map?, :one-MapEntry? #(= 1 (count %))}})
;; => true
(valid-collections?
  (list 11 (list 22 33))
  (list list? (list #(<= 2 (count %))) #(= 2 (count %))))
;; => true

Heterogeneous, nested collections work the same way.

(valid-collections?
  [{:a 1, :b 2} 33 {:d 4, :e 5} 66]
  [#(= 2 (count (filter map? %))) {:has? #(contains? % :b)}
   {:has? #(contains? % :d)}
   (fn [v]
     (every? int? (filter #((complement map?) %) v)))])
;; => true
(valid-collections?
  {:a [1 2 3], :b (list \z \q \w), :c {:x "abc", :y 42}}
  {:expected-count? #(= 3 (count %)),
   :a {:type vector?, :length #(<= 2 (count %))},
   :c {:contains? #(contains? % :x)},
   :b {:kind list?,
       :all-same? (fn [s] (apply = (map type s)))}})
;; => true
(valid-collections? [[[[]]]] [[[[empty?]]]]) ;; => true
(valid-collections? {:a {:b {:c {:d {}}}}}
                    {:a {:b {:c {:d {:contains-anything
                                       empty?}}}}})
;; => true

Sequence regexes

Replicating spec.alpha's regular expression facilities.

One or more integers, followed by zero or more strings.

(defn int+string*
  [s]
  (let [[front rear] (split-with int? s)
        ints #(<= 1 (count front))
        strs (every? string? rear)]
    (and ints strs)))
(int+string* [1 2 3 "a" "b"]) ;; => true(int+string* [1 2 3]) ;; => true(int+string* [1 2 3 'a 'b]) ;; => false
(valid-collections? [1 2 3 "a" "b"] [int+string*])
;; => true
(valid-collections? [1 2 3 'a 'b] [int+string*]) ;; => false

Exactly three keywords, followed by anything.

(defn three-kw-and?
  [s]
  (= 3 (count (take-while keyword? s))))
(valid-collections? [:a :b :c] [three-kw-and?]) ;; => true
(valid-collections? [:a :b] [three-kw-and?]) ;; => false
(valid-collections? [:a :b :c 1 2 3] [three-kw-and?])
;; => true

seq-regex, a little toy utility that streamlines this task. One or more integers, followed by zero or more strings.

(require '[speculoos.utility :refer [seq-regex]])
(valid-collections? [1 2 3 "a" "b" "c"]
                    [#(seq-regex % int? :+ string? :*)])
;; => true

Exactly three keywords, followed by anything.

(valid-collections? [:a :b :c 'foo "bar" \z]
                    [#(seq-regex % keyword? 3 any? :*)])
;; => true

See also Herbert and Seqexp.

Adjusting specifications

Speculoos specifications are just Clojure data structures, containing Clojure predicates. Adjust specifications with your favorite tools. clojure.core provides some nice ones.

(valid-collections? [[11] [22] [33]]
                    [[#(every? int? %)] [#(= 1 (count %))]
                     [#(not-empty %)]])
;; => false
(valid-collections? [[11] [22] [33]]
                    (drop-last [[#(every? int? %)]
                                [#(= 1 (count %))]
                                [#(not-empty %)]]))
;; => true
(valid-collections? {:a [1 2 3], :b ['foo 'bar 'baz]}
                    {:a [#(every? int? %)],
                     :b [#(every? int? %)]})
;; => false
(valid-collections? {:a [1 2 3], :b ['foo 'bar 'baz]}
                    (assoc-in {:a [#(every? int? %)],
                               :b [#(every? int? %)]}
                      [:b 0]
                      #(every? symbol? %)))
;; => true

Speculoos provides some specialized functions for manipulating heterogeneous, arbitrarily-nested data structures. Changing a predicate with assoc-in*.

(require '[speculoos.fn-in :refer
           [get-in* assoc-in* update-in* dissoc-in*]])
(valid-collections?
  [99 {:a 1, :b (list 42 22/7 ["abc" {:c [\z \q]}])}]
  [{:b (list [{:c [map?]}])}])
;; => false
(valid-collections?
  [99 {:a 1, :b (list 42 22/7 ["abc" {:c [\z \q]}])}]
  (assoc-in* [{:b (list [{:c [map?]}])}]
             [0 :b 0 0 :c 0]
             vector?))
;; => true

Removing a predicate with dissoc-in*.

(valid-collections? {:a [99 (list {:b [42]})]}
                    {:coll-test map?,
                     :a [(list {:b [map?]})]})
;; => false
(valid-collections? {:a [99 (list {:b [42]})]}
                    (dissoc-in* {:coll-test map?,
                                 :a [(list {:b [map?]})]}
                                [:a 0 0 :b 0]))
;; => true

Repairing data

Adjust or remove datums such that the data collection satisfies your specification. Using clojure.core functions.

(valid-collections? [[1] [2 3] [4 5 6]]
                    [[vector?] [vector?] [map?]])
;; => false
(valid-collections?
  (update-in [[1] [2 3] [4 5 6]] [2] #(zipmap [:a :b :c] %))
  [[vector?] [vector?] [map?]])
;; => true
(valid-collections? {:a 42, :b {:c 99}}
                    {:b {:has-anything? #(= 0 (count %))}})
;; => false
(valid-collections? (assoc-in {:a 42, :b {:c 99}} [:b] {})
                    {:b {:has-anything? #(= 0 (count %))}})
;; => true

Using Speculoos' fn-in* functions.

(valid-collections? [11 [22 33 [44 55 66 [77]]]]
                    [[[[list?]]]])
;; => false
(valid-collections? (update-in* [11 [22 33 [44 55 66 [77]]]]
                                [1 2 3]
                                #(into '() %))
                    [[[[list?]]]])
;; => true
(valid-collections? {:a 42, :b {:c {:d [99 0 'foo]}}}
                    {:b {:c {:d [empty?]}}})
;; => false
(valid-collections? (dissoc-in* {:a 42, :b {:c {:d [99]}}}
                                [:b :c :d 0])
                    {:b {:c {:d [empty?]}}})
;; => true

Comprehensive validation

Perform independent scalar and collection validations with a single function call. valid? returns true/false.

(require '[speculoos.core :refer [valid? validate]])
(def no-strings? #(every? (complement string?) %))
(valid? [42 :foo \z] [int? keyword? char?] [no-strings?])
;; => true
(valid? {:a 42, :b ['foo "abc"]}
        {:a int?, :b [symbol? string?]}
        {:has? #(contains? % :a), :b [#(<= 2 (count %))]})
;; => true

validate returns the validation report.

(validate [42 :foo \z] [int? keyword? char?] [no-strings?])

Instrumenting functions

Arguments and returns implicitly validated on every function invocation.

(defn multiplier
  {:speculoos/arg-scalar-spec [int? pos-int? ratio?],
   :speculoos/arg-return-spec [number? string?]}
  [x y z]
  [(* x y z) (str y)])
(multiplier 2 3 8/2) ;; => [24 "3"](multiplier 2 3 4) ;; => [24 "3"]
(instrument multiplier)(multiplier 2 3 8/2) ;; => [24 "3"](multiplier 2 3 4) ;; => [24 "3"]
(with-out-str (multiplier 2 3 4))
;; => ({:path [2], :datum 4, :predicate #function[clojure.core/ratio?], :valid? false})

(unstrument multiplier) ;; => {}(multiplier 2 3 8/2) ;; => [24 "3"](multiplier 2 3 4) ;; => [24 "3"]

Validating macros

Speculoos validates macros by applying specifications against the macro expansion.

(defmacro infix
  [form]
  (list (second form) (first form) (last form)))
(infix (2 + 3)) ;; => 5(macroexpand-1 '(infix (2 + 3))) ;; => (+ 2 3)
(def infix-spec (list symbol? int? int?))
(require '[speculoos.core :refer [valid-macro?]])
(valid-macro? '(infix (2 + 3)) infix-spec) ;; => true
(valid-macro? '(infix (2.2 + 3.3)) infix-spec) ;; => false

fn* and fn-in*

Starred functions inspect and manipulate values buried in heterogeneous, arbitrarily-nested data structures.

Inspecting values

(require '[speculoos.fn-in :refer
           [get-in* assoc-in* update-in* dissoc-in*]])
(get-in* [11 22 [33 44 55 [66 [77 [88 99]]]]] [2 3 1 1 1])
;; => 99
(get-in* {:a {:b {:c {:d 99}}}} [:a :b :c :d]) ;; => 99
(get-in* (list 11 22 33 (list 44 (list 55))) [3 1 0])
;; => 55
(get-in* #{11 #{22}} [#{22} 22]) ;; => 22
(get-in* [11 22 {:a 33, :b [44 55 66 {:c [77 88 99]}]}]
         [2 :b 3 :c 2])
;; => 99
(get-in* {:a (list {} {:b [11 #{33}]})} [:a 1 :b 1 33])
;; => 33

Associating values

(assoc-in* [11 [22 [33 [44 55 66]]]] [1 1 1 2] :new-val)
;; => [11 [22 [33 [44 55 :new-val]]]]
(assoc-in* {:a {:b {:c 42}}} [:a :b :c] 99)
;; => {:a {:b {:c 99}}}
(assoc-in* {:a [11 22 33 [44 55 {:b [66 {:c {:d 77}}]}]]}
           [:a 3 2 :b 1 :c :d]
           "foo")
;; => {:a [11 22 33 [44 55 {:b [66 {:c {:d "foo"}}]}]]}

Updating values

(update-in* [11 22 33 [44 [55 66 [77 88 99]]]]
            [3 1 2 2]
            inc)
;; => [11 22 33 [44 [55 66 [77 88 100]]]]
(update-in* {:a [11 22 {:b 33, :c [44 55 66 77]}]}
            [:a 2 :c 1]
            #(+ 5500 %))
;; => {:a [11 22 {:b 33, :c [44 5555 66 77]}]}

Dissociating values

(dissoc-in* [11 22 [33 [44 55 66]]] [2 1 1])
;; => [11 22 [33 [44 66]]]
(dissoc-in* {:a [11 22 33 {:b 44, :c [55 66 77]}]}
            [:a 3 :c 0])
;; => {:a [11 22 33 {:b 44, :c [66 77]}]}

Clamping non-terminating sequences

(require '[speculoos.utility :refer [clamp-in*]])
(clamp-in* [11 22 33 [44 55 [66 (repeat 42)]]] [3 2 1] 3)
;; => [11 22 33 [44 55 [66 [42 42 42]]]]
(clamp-in* {:a [11 22 33 {:b (cycle ['foo "bar" \z])}]}
           [:a 3 :b]
           5)
;; => {:a [11 22 33 {:b [foo "bar" \z foo "bar"]}]}

Advanced random sample generators

Built-in basic generator, set generator, and string regex generator.

(data-from-spec [int? #{:water :coffee :tea :milk}
                 #"fo{3,5}!"]
                :random)
;; => [-409 :tea "fooooo!"]

Custom generators attached to predicate's metadata.

(def manual-predicate
  (with-meta #(int? %)
    {:speculoos/predicate->generator #(rand-int 99)}))
(defpred easier-predicate #(int? %) #(rand-int 99))
(data-from-spec [manual-predicate easier-predicate] :random)
;; => [6 57]

Automatic random sample generators. or generators emit multiple types.

(defpred or-predicate
         #(or (int? %) (string? %) (keyword? %)))
(data-from-spec (repeat 5 or-predicate) :random)
;; => ["117sVCJ2Ams660hcBYioNN" :! :?Y.9s7S 24 29]

and refines a basic generator.

(defpred and-predicate
         (fn [i] (and (int? i) (odd? i) (<= 3 i))))
(data-from-spec (repeat 5 and-predicate) :random)
;; => [29 29 9 27 27]

Combining and/or.

(defpred and-or-predicate
         #(or (and (int? %) (pos? %))
              (and (string? %) (<= 3 (count %)))
              (and (ratio? %) (<= 1/9 %))))
(data-from-spec (repeat 4 and-or-predicate) :random)
;; => [7/2 23/25 "1eSSSlKXPMpN6o2Y4Z22yn0JDk" "baiPcIqp65QS4"]

Identifying predicates without generators.

(unfindable-generators
  [manual-predicate (fn no-gen [] (rand-int 99)) easier-predicate])
;; => [{:path [1], :value no-gen}]

Checking a generator against its host predicate.

(defpred pred-with-broken-gen #(neg? %) #(rand-int 99))
(validate-predicate->generator pred-with-broken-gen 5)
;; => ([14 false] [46 false] [57 false] [40 false] [12 false])

Predicates

Speculoos predicates are regular Clojure functions and compose the same way.

(valid-scalars? [42 "abc"]
                [#(and (int? %) (even? %))
                 #(and (string? %) (= 3 (count %)))])
;; => true
(valid-scalars? {:a 'foo} {:a #(or (char? %) (symbol? %))})
;; => true

clojure.core provides some nice higher-order functions for composing predicates: every-pred and some-fn.

(valid-scalars? [42 "abc"]
                [(every-pred int? even?)
                 (some-fn symbol? string?)])
;; => true

Placeholders, ambivalence

Maintain a specification's correspondence to a sequence.

(valid-scalars? [42 "abc" :foo] [int? symbol? keyword?])
;; => false
(valid-scalars? [42 "abc" :foo] [int? any? keyword?])
;; => true
(valid-scalars? [42 "abc" :foo]
                [int? (constantly true) keyword?])
;; => true
(valid-scalars? [42 "abc" :foo]
                [int? (fn [_] true) keyword?])
;; => true

Regular expression predicates

nil returned from a regular expression failure can indicate an invalid string.

(def example-regex #"\w{3}\d{3}")
(re-matches example-regex "abc123") ;; => "abc123"(re-matches example-regex "qr89") ;; => nil
(def regex-predicate #(re-matches example-regex %))
(valid-scalars? ["abc123" "xyz789"]
                [regex-predicate regex-predicate])
;; => true
(only-invalid (validate-scalars ["abc123" "qr89"]
                                [regex-predicate
                                 regex-predicate]))
;; => ({:datum "qr89",
;;      :path [1],
;;      :predicate #function [recipes/regex-predicate],
;;      :valid? nil})

Speculoos treats free-floating regexes as predicates.

(validate-scalars ["abc123" "qr89"]
                  [#"\w{3}\d{3}" #"[qrs]{2}[789]{2}"])
;; => [{:datum "abc123",
;;      :path [0],
;;      :predicate #"\w{3}\d{3}",
;;      :valid? "abc123"}
;;     {:datum "qr89",
;;      :path [1],
;;      :predicate #"[qrs]{2}[789]{2}",
;;      :valid? "qr89"}]

Sequence regexes

Mimicking spec.alpha's sequence regexes, use speculoos.utility/seq-regex during a collection validation.

(def three-ints-two-kw-and-syms?
  #(seq-regex % int? 3 keyword? 2 symbol? :*))
(valid-collections? [1 2 3 :a :b 'foo 'bar 'baz 'qux]
                    [three-ints-two-kw-and-syms?])
;; => true
(def sym-str-opt-int?
  #(seq-regex % symbol? :. string? :. int? :?))
(valid-collections? (list 'foo "abc" 42)
                    (list sym-str-opt-int?))
;; => true

Sets as predicates

Within a scalar specification, a set creates a predicate that tests for membership.

(valid-scalars? [:orange] [#{:red :orange :yellow}])
;; => true
(valid-scalars? [:pantheon] [#{:red :orange :yellow}])
;; => false
